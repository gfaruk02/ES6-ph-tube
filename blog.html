Var Scope:

Var is a JavaScript Variable declaration "keyword". It declarations are globally scoped or function scoped.
When var is declared outside function is called Global scope and this variable can be accessed anywhere in the program.
When var is declared inside function is called function scope and it's variable can be accessed in function. Var variables can be updated and re-declared within its scope;

Let Scope:

The scope of a let variable is block scope.It's declared in the '{}' and accessed only inside this block. let variables can be updated but not re-declared;

Const Scope:

The scope of a const variable is block scope. It's declared in the '{}' and accessed only inside this block. const variables can neither be updated nor re-declared.

The use cases of null and undefined

Undefined

Undefined is a special value in JavaScript that represents the absence of a value or the initial absence of a variable or property. It has several use cases in JavaScript

Variable diclaretion: When you declare a variable in your program ether global or local but don't assign a value to variable, it's automatically initialized to undefined.
Function Arguments: If you write a function is called with fewer arguments than it expects, the missing arguments are automatically set to undefined.
Object Properties: When you access an object property doesn't exists on an object will give you undefined.
Accessing array: accessing array elements outside of the index rang will give you undefined

Null

Null means an empty value or a blank value. The typeof() operator returns the type as an object for a variable whose value is assigned as null.
Default Values of Null:You want to use null as a default value for your function parameters


Initialize as null: When you set variable Initialize value as null, indicating no data yet
